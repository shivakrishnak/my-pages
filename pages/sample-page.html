§1What this proves

This page proves the minimum setup works: one script tag with the correct relative path, and a data-layout attribute. Everything else — stylesheet, sidebar, topbar, dark mode toggle, mobile bottom nav, the way back to the homepage — is generated automatically by core/init.js.

Component classes available out of the box: .ms-callout (this box), .ms-table, .ms-framework, .ms-steps, .ms-quote, .ms-drill, .ms-badge, .ms-checklist. See pages/fitness/workout-plan.html or any page in pages/mastery/ for live examples of each in a real, content-heavy page.

§2The three layout modes

Set on <body data-layout="...">:

Value What you get Use it for
full (default) Sidebar + topbar + bottom-nav Reference docs, anything with multiple sections
mixed Topbar + bottom-nav, no sidebar Single-screen tools, dashboards, short notes
none Nothing injected — just shell.css linked and dark mode applied Fully custom layouts that need complete control

Leaving the attribute off <body> entirely behaves exactly like full — so a brand new page with zero configuration still gets the complete hub experience.

§3Useful data attributes

Attribute Effect
data-title="..." Text shown in the topbar
data-badge="..." Small colored pill next to the title
data-badge-color="..." indigo · teal · amber · coral · sage
data-brand="..." Sidebar header text (full layout only)
data-toc="true" Auto-builds a sidebar table of contents from this page's <h2>/<h3> headings
